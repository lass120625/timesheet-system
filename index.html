🐙 GitHub Token 設定指南

步驟一：生成 GitHub Personal Access Token
  1. 前往 GitHub Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. 點擊 "Generate new token" → "Generate new token (classic)"
  3. 填寫：
    • Note: 工時管理系統
    • Expiration: No expiration（或選擇一年）
    • Scopes: 勾選 repo（Full control of private repositories）
  4. 點擊 "Generate token"
  5. 複製生成的 token（只會顯示一次！）
步驟二：在系統中設定 Token

步驟三：Repository 資訊

系統會自動使用你當前的 repository：

Repository: lass120625/timesheet-system
資料檔案: timesheet_data.json

💡 系統架構說明

GitHub 資料庫概念：

  • 主檔案：GitHub repository 中的 "timesheet_data.json"
  • 上傳流程：CSV → 解析 → 整合 → 推送到 GitHub
  • 同步機制：所有人共用同一個 repository
  • 版本控制：完整的 Git 歷史記錄

團隊使用方式：

  1. 主管設定好 GitHub Token 並分享網址給團隊
  2. 任何人都可以上傳工時 CSV
  3. 系統自動整合所有資料到 GitHub
  4. 其他人重新整理頁面就能看到最新資料
  5. 在 GitHub 上可以查看完整的更新歷史
