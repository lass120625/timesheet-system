🚀 快速部署指南

方案一：GitHub Pages（推薦）
  1. 建立 GitHub 帳號
  2. 建立新的 repository，命名為 "timesheet-system"
  3. 上傳這個 HTML 文件，命名為 "index.html"
  4. 在 Settings → Pages 中啟用 GitHub Pages
  5. 你的網站網址：https://你的用戶名.github.io/timesheet-system
方案二：Google Sites（最簡單）
  1. 前往 Google Sites
  2. 建立新網站
  3. 插入 → 嵌入 → 嵌入代碼
  4. 將整個 HTML 代碼貼入
  5. 發布網站，取得分享連結
Google Drive API 設定
  1. 前往 Google Cloud Console
  2. 建立新專案或選擇現有專案
  3. 啟用 Google Drive API
  4. 建立憑證 → OAuth 2.0 客戶端 ID
  5. 設定授權網域為你的網站網址

💡 系統架構說明

中央資料庫概念：

  • 主檔案：Google Drive 中的 "timesheet_database.json"
  • 上傳流程：CSV → 解析 → 整合 → 更新主檔案
  • 同步機制：所有人共用同一個主檔案
  • 版本控制：每次更新都保留時間戳記

團隊使用方式：

  1. 主管設定好系統並分享網址給團隊
  2. 任何人都可以上傳工時 CSV
  3. 系統自動整合所有資料到 Google Drive
  4. 所有人看到的都是最新的整合資料
  5. 可隨時下載完整資料檔案備份
